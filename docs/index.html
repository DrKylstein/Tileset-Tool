Tile Properties Dock

Return to Top

Gameplay properties of the currently selected tile, or the template tile in paint mode, are edited here. A change to a property will not take effect until the control looses focus. Hotkeys are also provided.

Animation Dock

Return to Top

This dock contains editing and playback controls for the animation of the current tile. The box on the left is a preview of the animation. The buttons beneath it can rewind, step back one frame, start and stop, step forward, and skip to the end of the animation, respectively. The box on the right shows the animation as a filmstrip. You can seek to a specific frame by clicking on it. By double-clicking on a frame in the filmstrip, you can change which graphic the frame uses. The dialog that opens will highlight the current graphic, and a single click on any other graphic cell will close the dialog, setting the frame to the clicked graphic. Next to the playback controls is a drop-down box that can be used to select a preset animation sequence. The animation will be set relative the current first frame when the Apply button is clicked, or the hotkey is activated.

Palette Dock

Return to Top

This dock displays the tileset's internal palette. This will be set automatically when importing graphics, and can also be imported or corrected with an external file. The palette is divided into 16 pages of 16 colors each. Only one page can be used by the game at a time. If you are using the palette-enabled tileset loader patch with the latest palette patch, the game will load all 16 pages into memory when a level is loaded, and the first page will be enabled. The remaining pages are for use by effect patches.

Paint Mode

Return to Top

Paint mode can be toggled from the paintbrush icon on the toolbar. While in paint mode, the properties and/or animation of the last tile selected will be applied to every new tile selected. Animations will be adjusted automatically so that the offset between the first-frame of the new tile and the first-frame of the template tile will be the same as the offset between the new tile and the old-tile in the tileset.

Whether paint mode copies properties, animations, or both can be set from the drop-down menu on the toolbar button, or from the edit menu.

Tools

Return to Top
Import image...

Load tile graphics and palette from an image.

The image must be in a supported format and divide evenly into 16x16 pixel cells. This action cannot be undone.

Fix palette...

Remap the tile graphics to a specified palette/

You wil be asked whether to use the standard EGA palette. This is the palette used by the Commander Keen games in the unmodified form. If you choose Yes, the tileset will load 16 identical pages of this palette. If No, see Import palette... for details. Once the palette is loaded, the colors of the tile graphics will be automatically re-mapped to their closest match in the new palette. This action cannot be undone.

Import palette...

Replace the current palette with one loaded from a file.

A palette file will be loaded directly. For an image file, if it has an internal palette, that will be used. Otherwise, the top-leftmost 16x16 pixel square will be interpreted as a palette. If the file does not contain 256 colors, or is shorter than 16 pixels for True Color images, the first 16 colors found will be mapped to the remaining color pages. Unlike Fix palette..., this operation will not adjust the colors of the tile-graphics--they will use whatever color now occupies the position of the previous one.

Auto-map tiles

Write all of the animations in the file such that tile 1 references tile graphic 1, and so on.

This is provided as a convenience for the inital layout of a tileset, and for those used to TileInfo and earlier versions of Tileset Tool. It may be very slow, and cannot be undone.

Export image...

Save tile graphics to an image.

This operation exists as counterpart to Import Image. It can be used to recover the tile graphics data for editing if the source has been lost. The resulting file will be a vertical strip of tiles from first to last as stored in the tileset data. For a more neatly arranged file, see Export to External Editor.

Export to external editor...

Save mapped tile graphics to an image.

Unlike Export Image..., this operation will create an image identical to the one seen in the main editor (minus scaling). This is suitable for use in level editors which do not support the tileset patch, or as a placeholder tileset file for modkeen-like utilities.

Export palette...

Save palette to an external file.

This operation exists as counterpart to Import palette... It will save to a palette format which may be usable with a non-tileset aware palette patch.

Supported Formats

Opening Tilesets
  • Lemm's tileset format (*.tls)
  • TileInfo-style binary patch (*.tli)
Saving Tilesets
  • Lemm's tileset format (*.tls)
Image formats
  • Windows Bitmap (*.bmp)
  • Portable Network Graphics (*.png)
  • Tagged Image Format (*.tiff)
Palette formats
VGA RAMDAC register dump file (*.bin, *.dat)
